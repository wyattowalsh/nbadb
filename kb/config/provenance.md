# Provenance Contract

## Purpose
This file defines the minimum `provenance` requirement for maintained KB notes.

## Minimum rule
Every maintained `wiki` page with substantive claims must include a `Provenance` section that links each claim cluster back to:
- `raw`
- or declared `canonical material`

## Preferred references
- repo file paths in code ticks for internal canonical sources
- stable note links for other vault notes
- external URLs only when the corresponding `raw` capture does not exist yet

## Source-summary rule
Every source-summary note must include:
- a `Source record`
- a concise `Summary`
- a `Provenance` table
- planned `wiki` coverage

## Drift rule
If counts or wording disagree across canonical sources, preserve the disagreement explicitly until the repo reconciles it.
