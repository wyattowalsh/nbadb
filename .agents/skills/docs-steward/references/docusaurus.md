# Docusaurus Reference

## 1. Version Snapshot (2026-03-05)

- `@docusaurus/core`: `3.9.2`
- `@docusaurus/preset-classic`: `3.9.2`

Re-check before claiming "latest": see `version-refresh.md`.

## 2. Core Project Signals

- `docusaurus.config.js` or `.ts`
- `sidebars.js` or `sidebars.ts`
- `docs/` folder with frontmatter-driven pages

## 3. High-Value Sync Workflow

1. Validate routes/baseUrl and i18n settings.
2. Validate sidebar graph against file tree.
3. Build docs and inspect route/link warnings.
4. Check for stale autogenerated sidebars.

## 4. Advanced Configuration Focus

- Keep docs plugin options explicit (path, routeBasePath, sidebarPath).
- Use doc tags and category metadata consistently.
- Ensure blog/docs separation remains clear.
- Keep versioned docs strategy explicit if using versions.

## 5. Health Checks

- Broken markdown links and route collisions
- Sidebar items pointing to missing IDs
- Undocumented pages outside navigation
- Theme/plugin incompatibilities after upgrades

## 6. Enhancement Priorities

- Use docs landing pages with intent-based pathways.
- Add "next step" links for tutorial continuity.
- Normalize admonition usage and code language tags.
- Reduce content duplication between docs and blog.

## 7. Common Anti-Patterns

- Overloaded sidebars with mixed audience depth
- Unversioned breaking changes in docs
- Silent route shifts from config edits
- Inconsistent heading IDs causing anchor drift

## 8. Command Skeletons

```bash
# Install/update
pnpm add @docusaurus/core@latest @docusaurus/preset-classic@latest

# Build checks
pnpm docusaurus build
pnpm docusaurus swizzle --help
```

Use project scripts when available.
